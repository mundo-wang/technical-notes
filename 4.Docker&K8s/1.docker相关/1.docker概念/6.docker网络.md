docker的网络模式是用来解决容器之间以及容器与外部世界之间的网络通信和隔离问题，不同的网络模式提供不同程度的隔离和连接性，以适用各种应用场景。

### 桥接模式

桥接模式是docker**默认**的网络模式，每个容器连接到一个本地的虚拟桥接网络。

桥接网络会在主机上创建一个虚拟网桥，容器都连接到这个网桥上。

每个容器都有一个唯一的IP地址，且可以通过这个IP地址与容器进行通信。

### 主机模式

容器直接使用宿主机的网络命名空间，和宿主机共享网络。

容器使用宿主机的IP和端口，无需进行端口映射。

容器和宿主机的网络完全一致。

使用主机模式时，需要用额外的参数配置：

```shell
docker run -d --name mycontainer --network host myimage
```

### 自定义网络

docker默认有一个桥接网络和一个主机网络，我们还可以创建自己的自定义网络。

既然有了这么两个网络，为什么还要再创建自定义网络？创建自定义网络提供了更好的隔离和安全性，使得容器之间通过网络通信，降低了横向攻击的风险。同时，它启用了容器名称的内部DNS解析，简化了容器间的通信和管理。自定义网络还允许灵活设计容器之间的网络拓扑，提高了性能和扩展性。

具体的创建命令，看“docker操作”部分。
