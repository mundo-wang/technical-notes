Kubernetes中的Node节点是集群中的工作单元，它是集群中的一台物理机器或虚拟机器。Node节点负责运行应用程序的工作负载，并由Master节点进行管理。

一个Node节点可以运行多个pod。

### Kubelet

运行在每个Node节点的主要组件，负责与Master节点通信，确保其所在节点上的容器都处于健康状态。

kubelet负责监视运行在节点上的pod，确保他们按照预期状态执行。

kubelet的工作流程：

1. 定期与Master节点的Api Server通信，获取pod和Node的信息。
2. 根据从Api Server获取的信息，确保节点运行了所需的pod
3. 启动和监控容器的运行状态。负责执行容器的健康检查。
4. Kubelet 定期监控节点上的资源使用情况，并将信息报告给 Master 节点。

### kube-proxy

是运行在每个Node节点上的网络代理，负责维护节点的网络规则，以及实现service的网络代理。

主要完成的任务：

1. 服务代理和负载均衡（轮询、随机选择等算法）
2. 端口转发：可以把请求从service的集群IP转发到pod的IP和端口。
3. 负责节点间的通信
4. 动态更新网络规则

### Pod

pod是k8s中可运行一个或多个容器的最小执行单位，也叫最小部署单元。

pod里的容器共享网络和存储，并在同一主机上运行。

1. pod内的容器共享相同的网络命名空间，内部可以通过localhost互相通信。
2. pod内的容器可以共享相同的存储卷，支持协同工作。
3. pod的生命周期和其内部所有容器的生命周期相同，当pod内所有容器都终止时，pod就完成了它的生命周期。pod是原子的，要么完全运行，要么完全失败。

### 总结

Node节点与容器的关系是：一个Node节点可以有多个Pod，一个Pod可以有多个容器。
