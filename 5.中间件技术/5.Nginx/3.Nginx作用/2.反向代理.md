`Nginx`的`location`块中的`proxy_pass`关键字用于将特定路径反向代理到后端服务，配置如下所示：

```nginx
server {
    listen 8081;
    server_name 123.60.188.152;

    location /api/ {
        proxy_pass http://123.60.188.153:8082;
    }
}
```

这表示一旦请求路径以`/api/`开头，就将服务转发到`http://123.60.188.153:8082`。

例如前端的请求为：

```http
http://123.60.188.152:8081/api/user/info
```

路由到指定`Nginx`路径后，`Nginx`会将请求自动转发成：

```http
http://123.60.188.153:8082/api/user/info
```

转发后，对应的后端服务处理该请求，并把响应回传给前端。这样可以让前端和后端请求看起来同源，避免浏览器的跨域问题，且避免前端把后端真实`IP`与端口暴露给用户。

需要注意，当`proxy_pass`的目标`URL`末尾带有斜杠时，例如`http://123.60.188.153:8082/`，`Nginx`会将与`location`匹配的路径部分从原始请求中移除，然后将剩余路径拼接到目标`URL`后再进行转发。也就是说，转发后的请求将变为：

```http
http://123.60.188.153:8082/user/info
```

前端向`Nginx`服务器发送的带有`/api/`路径的请求都会匹配到该路由，并转发到指定的后端域名，由对应的服务处理。因此，在实际部署中，前端和后端可以部署在不同的服务器上，前端只需调用`Nginx`服务即可完成对后端服务的调用。

在`proxy_pass`指令下，可以通过`proxy_set_header`指令，指定请求转发至后端服务器时，`Nginx`需要携带的`HTTP`请求头。实际使用中，通常会统一配置三个较为常见且通用的请求头：

```nginx
location /api/ {
    proxy_pass http://123.60.188.153:8082/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

其中，以`$`开头的标识符均为`Nginx`内置变量，其取值主要来源于`Nginx`运行时的内部状态、客户端请求信息，以及请求在代理链路中的上下文信息等。上面的三个请求头的含义如下：

- `Host`：定义为`$host`，用于将请求头中的`Host`字段设置为客户端请求的主机名，即`server`块中路由匹配的地址。
- `X-Real-IP`：定义为`$remote_addr`，用于在请求转发时携带真实的客户端`IP`地址。
- `X-Forwarded-For`：定义为`$proxy_add_x_forwarded_for`，用于维护请求代理链，其格式如下：

```sh
X-Forwarded-For: 客户端IP, 上一级代理IP, Nginx服务器IP
```

这意味着，即使请求经过多层代理，后端服务器仍然能够通过该头信息追踪到最初发起请求的客户端`IP`地址。

假设客户端`192.168.1.10`通过`123.60.188.152`访问`Nginx`，而`Nginx`将请求反向代理转发到后端服务`123.60.188.153:8082`，后端服务器`123.60.188.153`的对应接口接收到的请求头大致如下：

```sh
Host: 123.60.188.152
X-Real-IP: 192.168.1.10
X-Forwarded-For: 192.168.1.10, 123.60.188.152
```
