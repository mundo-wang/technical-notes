学习ES以前，我们要先知道什么是正排索引和倒排索引。

### 正排索引

正排索引是一张将文档编号映射到文档内容的映射表，每个文档都有一个唯一的编号，编号可以是整数或者其他形式的标识符。通过正排索引，可以快速根据文档编号获取文档内容。

正排索引可以直接根据文档编号快速获取文档内容，适用于需要展示整个文档的场景。

假设有三个文档：

文档编号：1，内容：Go语言是一门强类型、静态类型的编程语言。

文档编号：2，内容：Go语言的并发模型以goroutine和channel为基础，简洁而强大。

文档编号：3，内容：Go语言通过垃圾回收机制管理内存，提供了简单而高效的内存管理。

| 文档编号 | 文档内容                                                     |
| -------- | ------------------------------------------------------------ |
| 1        | Go语言是一门强类型、静态类型的编程语言。                     |
| 2        | Go语言的并发模型以goroutine和channel为基础，简洁而强大。     |
| 3        | Go语言通过垃圾回收机制管理内存，提供了简单而高效的内存管理。 |

我们可以通过文档编号，直接获取文档的内容。

### 倒排索引

倒排索引以词汇为单位，对于每个词汇，都记录了包含该词汇的文档列表，以及每个文档中该词汇出现的位置。这样，搜索引擎可以根据用户查询的关键词快速找到包含这些关键词的文档列表。

倒排索引适用于文本检索，能够快速定位包含特定关键词的文档，是搜索引擎中常用的索引方式。

还是以上面的三个文档为例，倒排索引就可以表示为：

| 关键词    | 包含该关键词的文档列表 |
| --------- | ---------------------- |
| Go语言    | 1, 2, 3                |
| 强类型    | 1                      |
| 静态类型  | 1                      |
| 并发模型  | 2                      |
| goroutine | 2                      |
| channel   | 2                      |
| 垃圾回收  | 3                      |
| 内存管理  | 3                      |
| 简洁      | 2, 3                   |
| 高效      | 3                      |

通过倒排索引，可以根据用户查询的关键词快速找到包含这些关键词的文档列表。例如，如果用户搜索“Go语言 并发”，可以从倒排索引中迅速得到文档2，其中包含了与查询相关的内容。

关于分词的一些注意事项：

中文分词至关重要，因为中文不像英文一样有清晰的单词边界。通常我们会借助中文分词工具（例如jieba分词）将文本切分为有意义的词语。

在英文文本中，单词之间一般以空格或标点符号分隔，因此可以通过空格、标点等简单划分。但需留意一些特殊情况，如词干化（stemming）和词形还原（lemmatization）等处理，以将单词还原为基本形式。

分词过程中需要关注停用词处理，这些是在搜索引擎中通常会被忽略的常用词汇，如“的”、“是”、“在”等。这些词对搜索没有实际帮助，在建立倒排索引时会被过滤，以减小索引体积。同时，需考虑模糊匹配、同义词处理、词干提取等场景，以提高搜索的准确性。

对于关键词的匹配，并不是完全一致才会被匹配上。例如我们搜索“内存”，同样可以匹配上“内存管理”这个关键词。同时，使用同义词也可能会搜索到这个关键词。
