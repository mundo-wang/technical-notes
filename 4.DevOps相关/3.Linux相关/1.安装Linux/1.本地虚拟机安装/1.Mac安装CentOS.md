在`Mac`上，可以使用`UTM`软件创建`Linux`虚拟机，其官方下载地址为：https://mac.getutm.app/。

我们安装`CentOS-Stream-9-latest-aarch64-boot.iso`这个镜像，其下载地址如下：https://mirrors.tuna.tsinghua.edu.cn/centos-stream/9-stream/BaseOS/aarch64/iso/CentOS-Stream-9-latest-aarch64-boot.iso。

如果`boot.iso`镜像有问题，可以选择使用`DVD`镜像，其下载地址如下：https://mirrors.tuna.tsinghua.edu.cn/centos-stream/9-stream/BaseOS/aarch64/iso/CentOS-Stream-9-latest-aarch64-dvd1.iso

该系统基于`CentOS 8`或`RHEL 8`及以上版本，已不再采用传统的`network.service`进行网络管理，而是默认由`NetworkManager`统一接管。因此，系统中不存在`network.service`，也无需再通过`/etc/sysconfig/network-scripts`目录下以`ifcfg-`开头的配置文件来管理网络接口。在这里我们直接使用外部`SSH`工具连接上面查询到的`IP`地址即可。

使用`Royal TSX`连接该服务器，如下所示：

<img src="image/image-20260227124010953.png" alt="image-20260227124010953" style="zoom:40%;" />

在这里我们配置一下`DNS`为公共`DNS`，执行以下命令：

```sh
sudo cp /etc/resolv.conf /etc/resolv.conf.bak
echo -e "nameserver 8.8.8.8\nnameserver 1.1.1.1" | sudo tee /etc/resolv.conf
```

接着我们执行以下命令，看是否可解析为公网`IP`，网络连通正常：

```sh
ping -c 4 mirrors.centos.org
```

出现如下结果，表示`DNS`服务器已配置成功：

<img src="image/image-20260227143505251.png" alt="image-20260227143505251" style="zoom:40%;" />

但是，`/etc/resolv.conf`是由`NetworkManager`生成的，重启网络或系统后会被覆盖，所以我们要做长期配置。

首先确认该服务器的网络连接名称，执行如下命令：

```sh
nmcli connection show
```

命令执行输出如下所示：

<img src="image/image-20260227143705743.png" alt="image-20260227143705743" style="zoom:40%;" />

这表示当前使用的连接名是`enp0s1`，下面做持久化配置，让`NetworkManager`不再自动获取`DNS`，固定使用`8.8.8.8`和`1.1.1.1`。我们按顺序执行以下命令：

```sh
nmcli connection modify enp0s1 ipv4.ignore-auto-dns yes
nmcli connection modify enp0s1 ipv4.dns "8.8.8.8 1.1.1.1"
nmcli connection down enp0s1 && nmcli connection up enp0s1
```

执行完成后，再执行一下下面的命令，验证持久化是否正确：

```sh
cat /etc/resolv.conf
nmcli device show enp0s1 | grep IP4.DNS
```

展示出如下结果，表示已配置成功：

<img src="image/image-20260227143930632.png" alt="image-20260227143930632" style="zoom:40%;" />

后续可以使用`reboot`命令重启机器，再执行上面的验证命令，看看是否为公共`DNS`。
