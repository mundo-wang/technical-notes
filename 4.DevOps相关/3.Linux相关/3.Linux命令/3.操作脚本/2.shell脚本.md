`Shell`脚本是由一系列`Shell`命令组成的文本文件，主要用于自动化执行系统操作。脚本文件通常以`.sh`为后缀，由`Shell`解释器按行解析并执行。`Shell`脚本常见的执行方式有两种：一种是通过`sh`命令执行，另一种是通过`./`直接执行。两者的核心区别在于是否遵循脚本开头的`shebang`（解释器声明，例如`#!/bin/bash`或`#!/bin/zsh`），区别如下所示：

- 使用`sh script.sh`执行脚本时，实际调用的是`/bin/sh`解释器，脚本内容会作为参数传递给它执行。由于解释器已经在命令行中被显式指定，系统不会再去解析脚本文件头部的`shebang`。
- 使用`./script.sh`执行脚本时，系统会根据脚本第一行的`shebang`来调用对应的解释器，脚本会在预期的环境下运行。

例如，如果脚本依赖于`bash`特有的一些语法或特性，却使用`sh script.sh`来执行，就可能因为`sh`与`bash`在兼容性上的差异而出错；而通过`./script.sh`方式执行时，系统会依据脚本开头的`#!/bin/bash`来调用`bash`解释器，使脚本能够按预期正常运行。

另外，通过`sh`命令执行脚本时，不要求脚本本身具备执行权限；而使用`./`方式直接执行脚本时，脚本必须具有可执行权限，否则会报`Permission denied`错误，因此需要先通过`chmod +x script.sh`命令为脚本赋权：

```sh
chmod -x restart.sh   # 移除执行权限
sh restart.sh         # 可以正常执行
./restart.sh          # 会报错：Permission denied
```

需要注意的是，`shebang`必须位于脚本的第一行，并且行首不能包含任何空格或注释，否则系统将无法正确识别并解析解释器。也就是说，`#!`必须作为脚本文件的前两个字符出现。

若要在`Go`中执行外部命令（包括`sh`脚本或单条系统命令），通常可以通过标准库`os/exec`来完成。
