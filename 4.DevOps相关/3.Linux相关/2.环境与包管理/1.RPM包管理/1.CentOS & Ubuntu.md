常见的`Linux`发行版主要分为`Debian`系和`Red Hat`系。`Debian`系以`Debian`为上游，`Ubuntu`是在其基础上演进的常见下游发行版。`Red Hat`系以`RHEL`为上游，早期的`CentOS`基于`RHEL`稳定版本源码重建，长期作为与`RHEL`保持二进制兼容的典型下游发行版，但转向`CentOS Stream`后不再承担该下游角色。

在`Linux`终端执行下面命令，可以查看系统的名称、版本等信息：

```sh
cat /etc/os-release
```

若为`CentOS`，其示例输出如下：

```sh
NAME="CentOS Linux"
VERSION="7 (Core)"
ID="centos"
ID_LIKE="rhel fedora"
VERSION_ID="7"
```

若为`Ubuntu`，其示例输出如下：

```sh
NAME="Ubuntu"
VERSION="22.04.1 LTS (Jammy Jellyfish)"
ID=ubuntu
ID_LIKE=debian
VERSION_ID="22.04"
```

在本笔记中，凡涉及`Linux`相关操作，均默认基于`CentOS`版本进行说明，所以在这一节我们了解下`CentOS`与`Ubuntu`的区别。

对于`CentOS`的安装我们已经非常熟悉。在安装过程中，系统会要求设置`root`用户密码。默认情况下，系统会启用`root`账户，初次登录时可以直接使用`root`账户登录。安装过程中可以选择创建普通用户，但这并非必选步骤。

而`Ubuntu`系统在默认安装时不会启用`root`账户。安装过程中必须创建一个普通用户，该用户默认属于`sudo`组（在`CentOS`中类似的组是`wheel`组），具备`sudo`权限，用于执行管理员级操作。`root`账户本身是存在的，但由于未设置密码，无法直接登录。因此，系统初次登录只能使用普通用户账号，并通过`sudo`来完成需要管理员权限的操作。

在桌面体验方面，`Ubuntu`对桌面用户更加友好，默认提供`GNOME`桌面环境以及相对完善、易用的图形化管理工具。而`CentOS`的默认安装通常以最小化服务器版本为主，桌面相关体验较弱，如需使用图形界面，通常需要后续手动安装桌面环境。

在包管理系统方面，`CentOS`使用`RPM`包格式，对应的包管理工具为`yum`（`CentOS 7`以下版本）或`dnf`（`CentOS 8`以上版本）；而`Ubuntu`则使用`DEB`包格式，对应的包管理工具为`apt`。两种包格式彼此不兼容，不能混合使用。

`CentOS`和`Ubuntu`默认的用户登录`Shell`都是`/bin/bash`，日常交互体验一致。二者的主要区别在于`/bin/sh`的实现：`CentOS`通常指向`bash`，而`Ubuntu`通常指向`dash`。因此，在编写系统脚本时需要注意`/bin/sh`在`Ubuntu`下不完全兼容`bash`语法。

更多差异点将在后续实际使用和对比过程中逐步补充说明。
