`RPM`体系的核心在于`.spec`文件。`rpmbuild`在执行构建时会先读取`.spec`文件中的元数据与各阶段定义，然后按顺序执行对应的阶段脚本，最终生成`.rpm`包。`.spec`文件通常包含以下几个部分：

1. 头部信息区：用于定义软件包的元数据，包含`Name`、`Version`、`Release`、`Summary`、`License`等内容。
2. 描述区（`%description`）：用于提供软件包的详细说明。
3. 准备阶段（`%prep`）：用于解压源码和打补丁，常见命令为`%setup`与`%patch`。
4. 构建阶段（`%build`）：用于执行编译操作，通常包含`make`、`./configure`、自定义构建脚本等。
5. 安装阶段（`%install`）：用于将构建产物安装到`RPM`构建根目录`%{buildroot}`。
6. 文件列表区（`%files`）：定义最终打包进`RPM`中的文件，常见标记为`%doc`、`%dir`、`%config`等。
7. 脚本钩子区（可选）：用于在安装、升级、卸载时执行脚本，包括`%pre`（安装前）、`%post`（安装后）、`%preun`（卸载前）、`%postun`（卸载后）、`%trigger`（触发器脚本）。
8. 变更日志（`%changelog`）：记录版本更新历史。

传统方式生成`RPM`包，通常需要手动编写`.spec`文件，再通过`rpmbuild`进行构建。然而，由于`.spec`语法较为陈旧，学习成本较高且可维护性较差，这种方式如今已很少被采用。

可以使用`goreleaser`，这是目前`Go`项目发布`RPM`包的主流方案。可通过以下命令下载该工具：

```sh
go install github.com/goreleaser/goreleaser/v2@latest
```

执行完成后，使用下面命令验证是否下载成功：

```sh
goreleaser --version
```

`goreleaser`内部实际上调用的是`nfpm`进行打包构建。`nfpm`的`GitHub`仓库地址为：https://github.com/goreleaser/nfpm。

可以通过以下命令直接下载`nfpm`工具：

```sh
go install github.com/goreleaser/nfpm/v2/cmd/nfpm@latest
```

下面说明在什么情况下应当打`RPM`包，什么情况下应当构建`Docker`镜像。一句话概括二者的核心区别：`RPM`包是将程序安装到操作系统中，而`Docker`镜像则将应用及其运行环境一并封装；前者依赖宿主机环境运行，后者自带完整运行环境。

当软件需要被安装进操作系统，并由操作系统负责其生命周期管理，即软件需要成为`OS`的一部分时，应选择`RPM`；否则，更适合使用`Docker`镜像进行交付与部署。
