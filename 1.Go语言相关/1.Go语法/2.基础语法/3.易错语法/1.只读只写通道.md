我们在学习Go语言的通道时，会接触到只读通道和只写通道的概念，例如`<-chan int`表示一个只读通道，`chan<- int`表示一个只写通道。

但是不知道你有没有思考过，创建一个只读或者只写的通道有什么意义呢？只读通道只能从中读取数据，那么读取出来肯定是空值，因为你没机会往里写入数据。而只写通道只能往里写入数据，而不能读取，及时写进去了，也没有任何意义。

所以，在 Go 语言中，设置只读或只写通道的主要目的是对通道的一种使用限制。例如一个函数可以设置只读通道作为参数，这样，函数只能对传入此函数的通道做读取操作，如果尝试写入，就会有编译错误。

```go
func ReadFromChannel(c <-chan int) {
	//c <- 20  尝试写入，会报编译错误
	value := <-c
	fmt.Println(value)  // 打印30
	
}

func main() {
	c := make(chan int, 3)
	c <- 30
	ReadFromChannel(c)
}
```
