首先我们需要知道什么是逃逸。逃逸（Escape）通常指变量或对象的生命周期延长至超出其原始作用域，在Go语言中，逃逸指局部变量在函数结束后，依然被引用，导致这些变量的存储需要**从栈逃逸到堆上**。如果一个变量逃逸到堆上，它的生命周期延长至函数外部，可能会影响垃圾回收的行为和程序的性能。

例如下面这段代码：

```go
// 返回一个指向局部变量的指针
func createPointer() *int {
	x := 42
	return &x
}

func main() {
	// 调用 createPointer 函数，得到一个指向局部变量的指针
	pointer := createPointer()
	// 打印指针所指向的值
	fmt.Println(*pointer)
}
```

函数`createPointer`创建了一个局部变量x，返回了指向这个局部变量的指针。在main函数中，我们调用函数`createPointer`，得到了这个指针，并打印指针所指向的值。

逃逸分析的作用就是分析这个变量x有没有发生逃逸，在上面的代码例子中，逃逸分析器分析得出局部变量x的生命周期已经逃逸到了main函数中，因为main函数可以继续引用并使用这个指针，即便函数`createPointer`已经执行完毕了，所以它检测到这种情况，把局部变量x分配在堆上，以确保在 `main` 函数中仍然可以安全地引用它。

如果逃逸分析器分析到局部变量并没有逃逸，那么它会被继续分配在栈上。

逃逸分析的结果会影响编译器对内存的优化策略。通过将不逃逸的变量分配在栈上，可以避免不必要的堆分配，提高程序的性能。而如果发生逃逸，堆分配则是为了确保变量在函数返回后仍然可用。

**逃逸的类型：**

- **栈逃逸：** 当一个局部变量在函数内部分配，但由于其引用传递到了函数外部，这就是栈逃逸。编译器可能会决定将这个变量分配在堆上，以确保其在函数返回后仍然有效。
- **堆逃逸：** 当一个变量分配在堆上并且其引用没有离开分配它的函数，这被称为堆逃逸。这种情况下，编译器可能会决定将变量分配在栈上，避免堆分配的开销。

**逃逸分析的优化目标：**

- **减少堆分配：** 通过将一些对象分配在栈上而不是堆上，可以减少垃圾回收的负担，提高程序的性能。
- **减少锁的竞争：** 逃逸分析还可以用于减少锁的竞争。在某些情况下，将对象分配在栈上可以减少对共享资源的争夺，提高并发程序的性能。
