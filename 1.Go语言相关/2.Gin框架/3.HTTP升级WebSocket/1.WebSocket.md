WebSocket的中文名叫：网络套接字。

下面是关于WebSocket的几点定义：

1. WebSocket属于应用层协议，建立在传输层协议TCP之上，客户端和服务端通过一个初始的握手建立连接，这个握手过程是基于HTTP协议的。
2. 在WebSocket上，数据可以双向流动，不受 请求-响应 模式的限制，服务器可以主动向客户端发送数据，而客户端也可以向服务器发送数据。
3. WebSocket的连接是持久的，连接一旦建立，会一直保持打开状态。
4. WebSocket协议较为轻量，头部信息较少，减少了通信的开销。



WebSocket和HTTP的区别如下：

| 特征     | HTTP                   | WebSocket                         |
| -------- | ---------------------- | --------------------------------- |
| 通信模型 | 请求-响应              | 全双工通信                        |
| 连接性质 | 无状态                 | 持久连接                          |
| 初始握手 | HTTP握手，每次通信独立 | 初始HTTP握手后切换为WebSocket协议 |
| 头部开销 | 较大                   | 相对较小                          |
| 端口     | 默认80                 | 默认80或443                       |
| 应用场景 | 传统请求-响应模式      | 实时、双向通信                    |



WebSocket的主要应用场景：

1. 实时聊天工具
2. 在线协作工具
3. 数据的实时更新
4. 推送服务
