在使用`Gorm`框架进行操作时，通常需要一个数据库连接对象，这里使用`db`表示。接下来，以此前在`dao`目录中定义的`UserInfo`结构体为例，介绍一些常见的`Gorm`操作。`UserInfo`结构体如下所示：

```go
type UserInfo struct {
	gorm.Model
	UserName string
	Email    string
	Age      int32
	Phone    string
}
```

首先，对于查询操作，可以使用`Raw`方法执行原生`SQL`语句，并将结果集映射到指定对象中：

```go
var users []dao.UserInfo
db.Raw("SELECT * FROM users WHERE age > ?", 28).Find(&users)
```

`db.Raw`用于执行查询操作，即运行那种可以返回结果集的`SELECT`类型的`SQL`语句。这里字符串里的` ? `表示占位符，类似于`Mybatis`框架的`#{}`，用于动态填充参数，可防止` SQL`注入。

这种方式显然不够方便。接下来，列出一些常用的`Gorm`查询相关`API`，首先是基础的查询语句：

```go
user := dao.UserInfo{}
db.First(&user, "user_name ILIKE ?", "%jack%")
db.Last(&user, "user_name = ?", "david_miller")
db.Take(&user, "age < ?", 18)
```

这三个方法的功能类似，都是用于查询符合条件的记录并返回一条结果：

- `Take`：不进行额外排序，返回第一条记录。
- `First`：按主键升序排序后，返回第一条记录。
- `Last`：按主键降序排序后，返回第一条记录。

在做查询结果集填充到对象的操作时，一定要保证对象是空的。

如果需要获取完整的结果集，应使用`Find`方法。该方法既支持查询单条记录，也支持查询多条记录：

```go
user := dao.UserInfo{}
userList := []dao.UserInfo{}
db.Find(&userList, "age > ?", 28)
db.Find(&user, "email = ?", "isabel.green@email.com")
```

与`Find`类似，`Gorm`还提供了`Scan`方法，用于将查询结果映射到指定结构体中，但它更加灵活，通常用于自定义字段或非表结构映射的场景。代码如下所示：

```go
var result struct {
	UserName string
	Age      int32
}

// 只查询部分字段，并映射到自定义结构体
db.Model(&dao.UserInfo{}).
	Select("user_name, age").
	Where("age > ?", 20).
	Scan(&result)
```

`Scan`不仅支持映射到结构体，还可以映射到切片、`map`以及基本数据类型。

上述方法都会返回一个`tx *DB`对象，该对象包含一个`Error`字段。对于`Take`、`First`和`Last`方法，当查询结果为空时，会返回类型为`gorm.ErrRecordNotFound`的错误。而`Find`与`Scan`方法在查询结果为空时不会返回错误，若需要判断是否查询到数据，可以通过检查`tx *DB`对象的`RowsAffected`是否为`0`来实现。

`Gorm`支持使用`Where`和`Not`方法构造条件查询，代码如下：

```go
db.Where("user_name ILIKE ? and age > ?", "%jack%", 25).First(userList)
db.Where("user_name ILIKE ?", "%jack%").Or("age > ?", 30).Find(&userList)
db.Not("age > ?", 30).Find(&userList)
```

这里的`Where`方法里面也可以使用表实体对象放封装的查询条件，如下所示：

```go
userList := []dao.UserInfo{}
searchCond := dao.UserInfo{
	UserName: "Alice",
}
db.Where(searchCond).Find(&userList)
```

这种将查询条件封装到实体对象中的方法，适用于多个非必传参数作为过滤条件的情况，因为它不会处理未传值（零值）字段。

`Gorm`支持链式查询，其中方法的作用见名知意，与`SQL`语句中相同：

```go
db.Select("user_name, age").Order("age desc").Limit(10).Find(&userList)
```

可以使用`Count`方法，查询结果集的数量：

```go
var count int64 // 必须是int64类型
db.Model(&dao.UserInfo{}).Where("age > ?", 30).Count(&count)
```

`Gorm`支持使用`Group`和`Having`两个方法完成分组操作：

```go
var result []struct {
    AgeGroup int
    Count    int
}

db.Model(&dao.UserInfo{}).
    Select("FLOOR(age / 10) * 10 AS age_group, COUNT(*) as count").
    Where("age >= ?", 30).
    Group("age_group").
    Having("COUNT(*) > ?", 1).
	Scan(&result)
```

这句查询语句对`UserInfo`表中的用户年龄进行分组，统计每个年龄段的人数。具体来说，将年龄按每`10`岁一个区间进行分组，并统计每个区间的人数。只有年龄大于等于`30`岁且该年龄段人数超过`1`的分组才会包含在结果中。

这里只是列举了一些主要的操作，意在展示`Gorm`执行查询的方式。
